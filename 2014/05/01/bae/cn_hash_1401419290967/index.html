函数柯里化初探

BAE上迁移啦。。。

前几天刷代码战争的题目，刷到一题关于函数柯里化的。
题目比较复杂，难度系数为kpu4,最后没有完全解出，但基本概念思路已经有了，出题者有点为了难度而难度的感觉。

题目示例为

function defaultArguments(func, params) {//我们要写的}

测试用例为：

function add(a, b) {return a + b;}
var add_ = defaultArguments(add, {b:9});
console.log(add_(10));  //输出19
console.log(add_(10, 5)); //输出 15
var add_ = defaultArguments(add, {c:9});
console.log(add_(10));  //输出 undefined

为了简化点，我把题目简化了一些，再原题中还要支持以下测试用例

var add_1 = defaultArguments(add_, {a:3, b:2})
add_1(10); //12
add_1(10, 5); //15

概念

首先了解一下函数柯里化的概念：

柯里化（currying）,是把接收多个参数的函数换成接收一个参数（最初函数的第一个参数），并且返回接受余下的参数而且返回新函数的技术（来自维基百科）

定义有点绕口，简而言之就是创建一个带有默认参数的新函数。

如下简单例子：

function curry(fn) {
    var args = Array.prototype.slice.call(arguments, 1);
    return function() {
        var innerArgs = Array.prototype.slice.call(arguments);
        var finalArgs = args.concat(innerArgs);
        return fn.apply(null, finalArgs);
    }
}

function add(a, b) {
    return a + b;
}

var newAdd = curry(add, 10);

newAdd(3); //输出13
newAdd(4，5); //输出14  ，注意这里就是和文首不一样的地方

明白柯里化了，来看下题目给的要求。

解题思路

1. 审题

题目应该说是函数柯里化的一个高级应用，defaultArguments函数接收两个参数，一个为func就是需要被柯里化的函数，params为一个对象。

这里和curry简单例子不同的地方就是params这个参数的复杂要求，也是造成题目难度的地方。题目要求能够购机params对象的属性名对应来处理是设置那个参数为默认。

举例：

function test(a, b) {
    console.log(b);
}
var test_ = defaultArguments(test, {a:1});
test_(2); //输出2
var test_ = defaultArguments(test, {b:1});
test_(2); //输出1

所以不能简单的使用上面的curry 函数进行处理，在处理前先要知道输入的对象属性是否对应函数的参数名。这里就需要获得待处理函数的参数名列表了，然后根据对应名称来处理默认参数。

2.解题

首先，需要对输入的函数进行处理，获得其形参名，这个需求一般很少有。。。我使用的方法是字符串处理

然后，查找params中是否有对应的形参名，若有则设置默认参数。

最后，返回函数中需要处理返回函数参数是否设置，是使用默认参数还是形式参数

3.代码

function defaultArguments(func, params) {
    var arr = {}, funcStr = func.toString();
    funcStr = funcStr.split(')')[0].split('(')[1].split(,);

4.扩展

原题更难，难度在于允许进一步柯里化，这时输入的params也是对象类型，可是心函数其实是没有形式参数列表的。所以需要在首次柯里化时，记录是否为首次柯里化并记录原始函数的参数列表，然后再进行柯里化时进一步判断处理。
