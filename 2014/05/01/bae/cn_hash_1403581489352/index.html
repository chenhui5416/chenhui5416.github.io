GIT 简单操作流程及笔记

BAE上迁移啦。。。

主要内容涉及实验室项目过程中需要加入新的成员时需要处理的git问题，以及一些简单的git操作笔记，先自己脑补git是什么，版本管理是啥，分支开发概念等。

加入版本库

在bash中执行 ssh-keygen,生成一个公钥.pub后缀名，将公钥复制给仓库管理员。创建过程中密码可以留空，这样就不用每次push和pull的时候都需要输入密码，假如不留空，请记住啊！

若使用github 则参考github上面的操作，若是私密项目，需要管理员设置你的权限。

另外，你可能需要加入多个git仓库，ssh-keygen -f newname 这个时候就需要生成新的keygen,然后配置.ssh下对应的config文件内容。

Host github.com

User youmail@mail.com

IdentityFile /c/Users/Administrator/.ssh/id_rsa_newname

可能会有问题，可能的解决方法就是先加新源地址（之前测试忘记了。回头找个新源测试下更新）

拷贝源代码

bash中进入自己想要放置源代码的目标文件夹，命令同linux cd

然后执行git clone 仓库地址 （文件名）其中文件名可选 ，若留空则默认为仓库中的名字，建议留空（个人觉得），仓库地址找项目负责人要。

工作流程

首先你需要了解分支开发原理，这样有助于开发使用（自己百度）

作为快速上手，还有用于简单开发或者个人小团队开发，你可以慢慢理解分支开发，但以下工作流程是需要的。

  1. 这里，我们抛开分支了，所以都在主分支也就是master下进行开发（这在小项目中做素材开发时，和后期需求更改时可以，但实在不推荐！）
    然后每次开发前 先git fetch 查看远程是否有更新（因为这种工作机制使用在我们项目后期，所以基本不会又同时更新源代码，或者造成冲突情况）
    若有更新则可以执行git merge origin/master，或者你直接执行执行git pull等效上面两步。

  2. 在最新的代码上进行修改相关需求修改意见

  3. 修改完成后，将修改内容提交到远程仓库。步骤为 git add -A(或者一一添加新的修改文件)，然后git commit -a -m "相关信息"，最后git push上传到远程仓库。其中相关信息务必要写清楚本次的修改概要，以免出错代码回滚时不好回滚。vim不支持中文输入，所以你可以配置git 的默认编辑器变为sublime 这样执行时使用 git commit -a 会自动弹出sublime然后你输入中文关闭对应文件就好。命令如下：git config --global core.editor "'c:/program files/sublime text 2/sublime_text.exe' -w"

其他，这个只是用来快速接手项目的！自己要去找资料了解git。

常用命令个人笔记(待不断丰富)

git status查看当前状态
git branch branchname新建一个分支
git branch -a 查看所有分支
git remote -v查看远程分支地址
git reset --hard HEAD~1 回滚到最近上一次提交，相当于上一次没有提交
