git 多个sshkey 管理 笔记

一个git终端（bash），链接多个git服务器，比如github、bitbucket、**、自己搭建的服务器啥的，这时候就需要多个sshkey，并需要一定的配置来管理这些东西。

在github上有生产sshkeys的介绍，链接

其实很多人写过啦！现场查也是很快的呀！可是我就是想在自己blog里再写一下，因为今天添加第三个sshkey的时候我又忘了。。。还看别人的博客，所以自己写一遍也不错，以后就看自己的，就是这个目的。

第一次或者只需要对应一个git服务器

第一次或者只需要一个对应一个git服务器，使用默认的就好了，方便快捷。你只要按照上面的github sshkeys 的文章链接处理就好。假如你连的不是github，是gitlab平台那么类似，要是自己内部搭建的git仓库服务器，没有这种可视化界面，那么就把生产的id_rsa.pub文件交给管理员加入到服务器中就好了,怎么测试是否添加成功，github的介绍页面都有说哦。ssh -T git@github.com(url 改成你对应的git服务器地址就好)。

第一次之后，要添加新的服务器sshkey

命令一样：

cd ~
ssh-keygen -t ras -C "you@mail.com"

然后对生成的sshkey进行命名，输出如下（不要直接跳过了—）

cd ~/.ssh
$ ssh-keygen -t rsa -C "you@mail.com"
Generating public/private rsa key pair.
Enter file in which to save the key (/c/Users/Administrator/.ssh/id_rsa): newname(重命名)

添加私鈅

ssh-add ~/.ssh/newname

然后在~/.ssh 目录文件下建一个config文件，用来告诉你本地git 对应的服务器用什么key来通信

#github
Host github.com
    HostName github.com
    User you@mail.com
    IdentityFile ~/.ssh/id_rsa
#内部服务器
Host 202.199.199.9
    HostName 202.199.199.9
    User you@mail.com
    IdentityFile ~/.ssh/id_lab_rsa
#gitlab
Host git.ustclug.org
     HostName git.ustclug.org
     User you@mail.com
     IdentityFile ~/.ssh/gitlabustc

好了就是这样。
